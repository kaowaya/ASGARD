# I5.1 微网能量管理系统 - 算法原理详解

## 1. 问题建模

### 1.1 决策变量

微网能量管理的核心是确定每个时间段的最优控制策略：

- **P_charge[t]**：储能充电功率 (kW)
- **P_discharge[t]**：储能放电功率 (kW)
- **P_grid_import[t]**：从电网购电功率 (kW)
- **P_grid_export[t]**：向电网卖电功率 (kW)
- **SOC[t]**：电池荷电状态 (0-1)

### 1.2 目标函数

多目标优化函数：

```
minimize: α × Cost + β × Carbon + γ × Reliability
```

其中：
- **Cost**：总电费成本
  ```
  Cost = Σ(P_grid_import[t] × Price[t] × Δt)
  ```

- **Carbon**：碳排放量
  ```
  Carbon = Σ(P_grid_import[t] × EmissionFactor[t] × Δt)
  ```

- **Reliability**：供电可靠性惩罚
  ```
  Reliability = Σ(Loss_of_Load[t] × Penalty)
  ```

## 2. 约束条件

### 2.1 功率平衡约束

每个时刻的功率必须平衡：

```
P_grid + P_pv + P_discharge = P_load + P_charge
```

### 2.2 SOC动态约束

SOC状态转移方程：

```
SOC[t+1] = SOC[t] + (P_charge[t] × η_charge - P_discharge[t] / η_discharge) × Δt / Capacity
```

其中：
- **η_charge**：充电效率
- **η_discharge**：放电效率
- **Capacity**：电池容量 (kWh)

### 2.3 电池物理约束

```
SOC_min ≤ SOC[t] ≤ SOC_max
0 ≤ P_charge[t] ≤ P_max
0 ≤ P_discharge[t] ≤ P_max
P_charge[t] × P_discharge[t] = 0  # 不能同时充放电
```

### 2.4 电网约束

```
0 ≤ P_grid_import[t] ≤ P_import_max
0 ≤ P_grid_export[t] ≤ P_export_max
```

### 2.5 需量约束

```
max(P_grid_import) ≤ Demand_Contract
```

## 3. 优化算法

### 3.1 混合整数线性规划 (MILP)

#### 数学模型

```
minimize: c^T × x

subject to:
A_eq × x = b_eq
A_ub × x ≤ b_ub
lb ≤ x ≤ ub
```

其中：
- **x**：决策变量向量
- **c**：成本系数向量
- **A_eq, b_eq**：等式约束（功率平衡、SOC动态）
- **A_ub, b_ub**：不等式约束（容量限制）
- **lb, ub**：变量上下界

#### 求解器选择

- **开源**：SciPy (linprog), CBC, GLPK
- **商业**：Gurobi, CPLEX, MOSEK
- **推荐**：Gurobi（性能最佳）

### 3.2 动态规划

适用于小规模问题：

```
V(SOC, t) = min {
    Cost(SOC, action, t) + V(SOC', t+1)
}
```

状态转移：
```
SOC' = f(SOC, action, t)
```

### 3.3 启发式算法

基于规则的控制策略：

**峰谷套利策略：**
```
if Price[t] == Valley_Price and SOC[t] < SOC_max:
    action = Charge
elif Price[t] == Peak_Price and SOC[t] > SOC_min:
    action = Discharge
else:
    action = Idle
```

**光伏消纳策略：**
```
P_net = P_pv - P_load
if P_net > 0 and SOC[t] < SOC_max:
    action = Charge(power = min(P_net, P_max))
```

## 4. 预测算法

### 4.1 负荷预测

#### LSTM模型

```python
model = Sequential([
    LSTM(128, input_shape=(lookback, features), return_sequences=True),
    Dropout(0.2),
    LSTM(64, return_sequences=False),
    Dropout(0.2),
    Dense(32, activation='relu'),
    Dense(1)
])
```

#### 特征工程

输入特征：
- 历史负荷（过去168小时）
- 时间特征（小时、星期、月、节假日）
- 气象特征（温度、湿度）
- 领先指标（生产计划）

### 4.2 光伏预测

#### 物理模型

```
P_pv = P_rated × (G / G_ref) × [1 - α × (T_cell - T_ref)]
```

其中：
- **G**：实际辐照度 (W/m²)
- **G_ref**：标准辐照度 (1000 W/m²)
- **α**：功率温度系数 (%/°C)
- **T_cell**：电池板温度
- **T_ref**：参考温度 (25°C)

#### 机器学习模型

- **输入**：天气预报（辐照度、温度、云量）
- **模型**：XGBoost / LSTM / Transformer
- **输出**：未来24-72小时光伏发电预测

## 5. 实时修正策略

### 5.1 滚动优化

```
for t in 0:rolling_interval:horizon:
    1. 获取最新状态（SOC、负荷、光伏）
    2. 更新预测（负荷、光伏、电价）
    3. 求解优化问题
    4. 执行第一个控制指令
    5. 移动时间窗口
```

### 5.2 反馈修正

```
Error = SOC_actual - SOC_predicted
SOC_corrected = SOC_predicted + Kp × Error
```

## 6. 性能优化

### 6.1 计算加速

- **变量维度缩减**：从96点降到24点
- **约束松弛**：放宽非关键约束
- **热启动**：使用上次解作为初值
- **并行计算**：多场景并行求解

### 6.2 内存优化

- **稀疏矩阵**：约束矩阵稀疏化
- **增量求解**：只更新变化的部分

scenarios = generate_scenarios(load_forecast, pv_forecast, n_scenarios=100)
results = [optimize(scenario) for scenario in scenarios]

Cycle_Life = Base_Cycle_Life × exp(-k × DOD)

```

Battery_Degradation_Cost = Σ(Throughput × Cost_per_kWh)
```
