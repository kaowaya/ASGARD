## 算法原理

真实车辆回传的运行数据充满噪声：
1. **采样率极度抖动**：可能10秒一次，也可能在无信号区几分钟没有数据，收到后集中乱序上报。
2. **传感器毛刺**：ADC 采样偶尔出现几十伏的尖峰。
3. **休眠断点**：车辆停车休眠后，电流电压瞬间掉线。

### 核心处理工作流 (Pipeline Steps)

1. **时序对齐与去重 (Time Alignment & Deduplication)**
   - 根据时间戳强制排序，剔除完全重复的帧。
2. **离群点剔除 (Outlier Rejection)**
   - 动态物理边界约束（由于同时兼容 EV 电芯 `~4.2V` 与 ESS Pack `~1000V`，边界默认非常宽容 `[0.0V, 1000V]`，支持计算时通过 CLI `--v-min / --v-max` 动态缩紧）。
   - 滑动窗口 Z-Score 或 MAD（绝对中位差）过滤突变毛刺。
3. **重采样与插值 (Resampling & Interpolation)**
   - 使用 Pandas 的 `resample` 功能，制定标准的时间网格。**核心原则：除非算力极其匮乏，否则务必重采样为 `1Hz (1S)`**。因为 10S 及更低频的数据会彻底抹除类似“析锂后松弛电压平台”这样的毫秒级/秒级高频电化学特征。
   - 线性插值（小断点 < 30s）与前向填充（休眠/待机状态）。长断点直接切断连续数据段。
4. **高阶特征衍生 (Feature Engineering)**
   - `dV/dt` (电压微分，用于捕捉内短路或析锂)
   - `dI/dt` (电流微分，用于识别阶跃工况)
   - `dQ` (区间积分充入容量统计)
   - `dT/dt` (温升速率)

## 工业级约束要求

与本地嵌入式 C++ 不同，云端的预处理代码**严禁使用低效的 for 循环去逐行检查数据**，必须全部依靠基于 numpy 矩阵掩码 (Masking) 或 Pandas 矢量化逻辑来释放服务器算力潜力。

---

## 关键技术亮点

1. **并行处理支持**: 纯无状态映射函数，支持大数据批处理 (Batched DataFrame)。
2. **标准化输出**: 把千奇百怪的厂商协议全部清洗成 ASGARD 标准模型。
3. **自动片段分割**: 检测到充放电反转或长跨度停车时，自动将单一超长数据分割为独立特征片段。
